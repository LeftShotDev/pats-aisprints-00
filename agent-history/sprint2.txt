--- POST-LAUNCH CLEANUP ---

USER: I am running into the following deploy error... [same esbuild/TypeScript
context — already resolved above]

USER: One more small cleanup. When switching between archive and active questions,
this seems to be using state management. I think it'd be more preferable to rely
on a query parameter in the URL instead. Use case: when you preview an individual
question and try to go back to the listing, you don't always go back to the same
place. Also, after logging in, could we have the initial page URL redirect directly
to the questions page?

ASSISTANT: Implemented two changes:

  1. URL-based library state (QuestionLibrary.tsx):
     - Replaced useState for activeTab, searchQuery, page, pageSize with
       useSearchParams + useRouter + usePathname from next/navigation
     - Added updateParams() helper to write changes back to the URL
     - Kept searchInput as local state only (for debounce buffer)
     - Added useEffect to sync input when user navigates Back/Forward
     - Pagination Previous/Next buttons updated to call updateParams
     - handleArchive/handleUnarchive use a refreshKey increment instead of
       directly calling fetchQuestions (since URL params didn't change)
     - Default values omitted from URL to keep it clean:
       tab=active, page=1, pageSize=25 are not written to the URL
     - Example sharable URL: /questions?tab=archived&search=gravity&page=2
     - Added Suspense boundary to app/questions/page.tsx (required by
       Next.js App Router when using useSearchParams)

  2. Home page redirect (src/app/page.tsx):
     - Replaced the default Next.js boilerplate page with a redirect to /questions
     - Unauthenticated visitors are caught by middleware and sent to /login

  Changelog entry added to MCQ PRD.

USER: One final small change. Remove the ability to add a question on the archive
page when there are no questions listed.

ASSISTANT: Updated EmptyState.tsx:
  - Added showCreateButton prop (defaults to true — no breaking change)
  - When false: button hidden, heading → "No archived questions",
    subtext → "Questions you archive will appear here."
  - When true (active tab): heading → "No questions yet",
    subtext → "Get started by creating your first question."

  Updated QuestionLibrary.tsx:
